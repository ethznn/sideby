# Code of Conduct

Sideby aims to be a practical, respectful, and technically rigorous open-source project.

## Expected Behavior

- Be respectful and constructive.
- Assume good intent, but ask for evidence when technical claims matter.
- Keep discussion focused on the project and the user problem.
- Welcome people with different levels of macOS, Swift, design, and QA experience.
- Give actionable feedback instead of personal criticism.

## Unacceptable Behavior

- Harassment, threats, insults, or discriminatory language.
- Sexualized language or unwanted attention.
- Publishing private information without permission.
- Repeated off-topic disruption.
- Bad-faith issue reports or pull requests.

## Enforcement

Maintainers may remove comments, close issues or pull requests, limit participation, or ban contributors who violate this code of conduct.

If you need to report conduct concerns, contact the maintainer through the repository owner's public GitHub profile or another private channel published by the project. Do not use public issues for sensitive reports.
